# @capsuletech/compliance

**Расположение:** `packages/system/compliance/`
**Зависит от:** `@babel/parser`, `@babel/traverse`, `@babel/types`

Статический чекер архитектурного регламента HCA. Реализация по [[004-compliance-linter|ADR 004]]. Превращает [[golden-rules|правила слоёв]] из текста в инвариант, проверяемый на каждый transform Vite.

## Что ловит

| Нарушение | Иконка | Описание | Пример |
|---|---|---|---|
| `upward-import` | ⬆️ | нижний слой импортирует верхний | `import x from '@features/auth/login'` в Entity |
| `horizontal-import` | ↔️ | элемент импортирует соседа по слою | `Entity.A` тащит `Entity.B` |
| `disallowed-import` | 🚫 | импорт не из allowlist данного слоя | `axios` в Controller (нужен только во Feature) |
| `side-effect-fetch` | 🌐 | вызов `fetch`/`axios`/`XMLHttpRequest` вне Feature | `await fetch(...)` в Controller |

Каждое сообщение содержит `file:line:column` + hint что делать.

## API

```ts
import {
  classify,
  extractGroup,
  check,
  formatViolations,
  RUNTIME_ALLOWED,
  LAYER_PREFIXES,
  CROSS_LAYER_ALLOWED,
} from '@capsuletech/compliance';

import type { Layer, IViolation, ICheckOptions } from '@capsuletech/compliance';
```

### `classify(absPath): Layer`

Определяет слой файла по абсолютному пути:

```ts
classify('/apps/sandbox/src/entities/_auth/loginForm.tsx');  // 'entity'
classify('/apps/sandbox/src/widgets/forms/_auth.tsx');        // 'widget'
classify('/packages/ui/src/components/button/button.tsx');   // 'system'
classify('/apps/sandbox/src/x.spec.ts');                     // 'test'
classify('/.capsule/registry/wrappers.ts');                  // null  (skip)
classify('/node_modules/...');                               // null  (skip)
```

### `check(absPath, code, opts?): IViolation[]`

Главный чекер. Парсит код через babel, обходит импорты и `CallExpression`, возвращает список нарушений (пустой если чисто).

```ts
const violations = check(filePath, code);
if (violations.length) console.error(formatViolations(violations));
```

Опции:
```ts
{
  extraAllowed?: Partial<Record<Layer, RegExp[]>>,  // расширить allowlist по слоям
  checkSideEffects?: boolean,                        // default: true
}
```

## Правила (allowlists)

Базовые runtime-разрешения (`RUNTIME_ALLOWED`):

| Слой | Разрешено |
|---|---|
| `entity` | `solid-js`, `@capsuletech/style`, относительные |
| `controller` | + `xstate`, `@xstate/solid`, `es-toolkit`, `@capsuletech/state`, `@capsuletech/router` |
| `feature` | + `@app/api/*`, `@app/services/*` (доменный API/services приложения) |
| `widget` | `solid-js`, `@capsuletech/ui`, `@capsuletech/style` |
| `page` | `solid-js`, `@capsuletech/ui`, `@capsuletech/style`, `@tanstack/solid-router` |
| `system` | без ограничений |
| `test` | без ограничений |

Cross-layer импорты по alias (`CROSS_LAYER_ALLOWED`):

| Слой может импортировать |  |
|---|---|
| Widget | `@entities/`, `@controllers/`, `@features/` |
| Page | `@widgets/` |
| Все остальные | ❌ ничего |

> [!info]
> `import type` всегда разрешён — типы не создают runtime-связей. Sandbox-файлы в HCA обычно вообще без runtime-импортов (auto-import закрывает Page/Widget/Entity/Controller/Feature).

## Vite-плагин

`packages/system/vite/src/plugins/compliance.ts`

```ts
import { plugins } from '@capsuletech/building.ts';

plugins.CompliancePlugin({
  mode: 'warn' | 'error',         // default: 'warn'
  extraAllowed: { feature: [/^@app\/payment\//] },
  checkSideEffects: true,
});
```

Подключён в `packages/builders/vite/src/defines/capsuleConfig.ts` (после `HMRWrappingPlugin`).

### Режимы

- **`warn`** (default): нарушения выводятся как warning, dev-server и build не валятся. Используем на rollout: видим картину, чиним по списку.
- **`error`**: нарушение валит билд / dev-сервер. Включаем когда репо стало чистым.

## Расширение под свой проект

Если приложение использует свою конвенцию для API/services:

```ts
plugins.CompliancePlugin({
  extraAllowed: {
    feature: [/^@my-domain\/api/, /^@my-domain\/services/],
  },
}),
```

## Пропускаются

- `node_modules/**`
- `.capsule/**` (autogenerated)
- `*.spec.ts(x)`, `*.test.ts(x)` (test mode — full pass)
- Файлы вне известных слоёв (system, скрипты, tooling)
